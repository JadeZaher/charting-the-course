# Change Order: Tile Architecture & Admin Refactor
**Date:** Jan 28
**Status:** High Priority / Immediate Execution

## 1. Executive Summary & Philosophy
The current implementation of "Badges" and "Profile Features" is too hardcoded and specific. We are pivoting to a **Generic Tile Architecture**.
* **Goal:** Decouple content from logic. A "Badge" should not be a database table; it should just be a "Tile" that looks like a badge.
* **Why:** Once features are hardcoded (like specific badge logic), they are hard to undo. We need a flexible system where all survey outputs—whether they are text, scores, or badges—live in a generic data structure.

---

## 2. Core Architecture Changes

### A. Data Storage ("The One Column")
* **Requirement:** Move away from scattered tables for user results.
* **New Structure:** We require a **single column** (JSONB) to store a user's profile tiles.
* **Survey Output:**
    * **Stop** using "Pass/Fail" logic for quizzes.
    * Survey results must process directly into a generic data table (the Tile column).
    * Any survey can generate any type of tile (Text, Chart, Badge).

### B. The "Tile" Concept
* **Badges:** Pause the current hardcoded badge development. Badges are now just a specific *type* of Tile.
* **Generalization:** We must build the "Tile" system first. Once built, we can verify if the specific badge system is even needed, or if the Tile system covers that use case entirely.

---

## 3. Profile UI Refactor

### A. Public Profile
* **Single Page View:** The public profile must be simplified to a **single scrolling page**.
* **Unified Grid:** All tiles (Badges, Generic Results, Alignment details) must sit on this single page.
* **Removals:**
    * No "Journey" tabs.
    * No "Connections" tabs.
    * "Alignment" is no longer a separate page; it is effectively a Tile mixed into the main profile view.

### B. User Dashboard
* **Teams:** Users should **not** see a "Teams" interface. Teams exist only for backend assignment logic (pushing quizzes).

---

## 4. Admin & Security Overhaul

### A. Permissions Migration
* **Requirement:** Migrate away from hardcoded "Roles" (e.g., `user.role === 'admin'`).
* **New System:** Implement **granular Permissions**. Admin capabilities should be defined by what they can *do*, not their label.

### B. User Lifecycle
* **Archiving:** Admins must have the ability to **Archive Users** (soft delete) rather than just permanently deleting them.

### C. "Ghost Mode" (Proxy Taking)
* **Proxy Submission:** Admins must be able to take a quiz *on behalf of* another user.
* **Visibility Rules:**
    * If an Admin takes a quiz for a user, the **Quiz itself** must be invisible to that user (they shouldn't see it in their "To-Do" list).
    * However, the **Result (Tile)** of that quiz must still be incorporated into the user's profile.

---

## 5. Scope Cuts & Paused Features
*To ensure focus on the Tile Architecture, the following are explicitly paused or removed:*
1.  **Gamification:** Pause **Levels**. Pause **XP Points**.
2.  **Maps:** Pause/Hide Map View.
3.  **Hardcoded Badges:** Stop development on specific badge tables; move to Tiles.

---

## 6. Implementation Priorities
*Execute in this strict order:*

1.  **Survey-to-Tile Pipeline:** Build the logic to convert SurveyJS output into the generic Tile format (removing Pass/Fail).
2.  **Tile System:** Build the rendering, visibility, and setup for the Tile Grid.
3.  **Public Profile:** Simplify the UI to the "One Page" layouts using the new Grid.
4.  **Admin & Permissions:** Refactor the User Management to use Permissions and add Archiving.
5.  **Profile Panels:** Finalize the internal view of the profile.
6.  **Cleanup:** Hide/Delete the paused features (XP, Levels, Teams UI).
